# Ejercicio 3 MCP - Convertir un PDF con MarkItDown MCP y comparar el contexto

## Objetivo del ejercicio

Comparar una tarea de analisis basada en texto pegado manualmente contra la misma tarea usando conversión estructurada desde PDF mediante un MCP de conversión documental.

## Lo esperado

Al terminar, deberias notar si la conversion via MCP deja un contexto mas usable y reusable que copiar texto manualmente. No importa solo el tamaño del input; importa tambien la legibilidad y la facilidad para volver a usar el contexto.

## Instrucciones

1. Usa este archivo PDF como entrada:
   - [temario_curso_github_copilot.pdf](/Users/haroldgomez/Library/CloudStorage/GoogleDrive-haroldg@gmail.com/My%20Drive/Material%20propio/Curso%20IA%20Github%20Copilot/CursoIA/clase%207/ejercicio_3_mcp/temario_curso_github_copilot.pdf)
2. Abre el PDF y copia manualmente un fragmento de 1 a 2 paginas que hable de objetivos o sesiones.
3. Sin MCP, pide a Copilot:
   - resumir el contenido;
   - listar 3 temas centrales;
   - proponer una estructura breve.
4. Guarda el resultado en `analisis_sin_mcp.md`.
5. Ahora usa el MCP de conversion documental disponible en tu entorno, por ejemplo MarkItDown MCP.
6. Convierte el mismo PDF a Markdown.
7. Guarda o revisa el Markdown generado.
8. Vuelve a pedir exactamente la misma tarea a Copilot, pero usando el Markdown convertido como contexto.
9. Guarda el resultado en `analisis_con_mcp.md`.
10. Compara y registra:
   - cual contexto fue mas facil de reutilizar;
   - cual respuesta fue mas ordenada;
   - si el markdown mejoro realmente el trabajo o solo cambio el formato.

## Entregable

- `analisis_sin_mcp.md`
- `analisis_con_mcp.md`
- nota comparativa breve

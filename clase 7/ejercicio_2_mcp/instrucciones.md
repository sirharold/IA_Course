# Ejercicio 2 MCP - Crear o redactar un issue desde notas locales

## Objetivo del ejercicio

Usar GitHub MCP para pasar de una descripcion local de un problema a un issue mejor estructurado y, si el entorno lo permite, crearlo en un repo de practica.

## Lo esperado

Al terminar, deberias entender la diferencia entre pedir a Copilot que redacte texto y usar MCP para convertir esa salida en una accion sobre GitHub. El valor no esta solo en escribir mejor, sino en conectar la tarea con un sistema real.

## Instrucciones

1. Abre el archivo [notas_bug_local.md](/Users/haroldgomez/Library/CloudStorage/GoogleDrive-haroldg@gmail.com/My%20Drive/Material%20propio/Curso%20IA%20Github%20Copilot/CursoIA/clase%207/ejercicio_2_mcp/notas_bug_local.md).
2. Lee el problema y no agregues informacion extra que no este en el archivo.
3. Sin MCP, pide a Copilot que redacte un borrador de issue con:
   - titulo;
   - problema observado;
   - pasos para reproducir;
   - impacto;
   - validacion esperada.
4. Guarda el resultado en `issue_borrador_sin_mcp.md`.
5. Ahora usa GitHub MCP.
6. Pide a Copilot que tome las mismas notas y prepare el issue para el repo de practica indicado por el docente.
7. Si el entorno lo permite, crea el issue real.
8. Si el entorno no permite crear el issue, al menos deja el payload final o borrador estructurado.
9. Guarda el resultado en `issue_borrador_con_mcp.md`.
10. Escribe una comparacion de 5 lineas:
   - que cambio al pasar de texto a accion;
   - que permiso o costo adicional aparecio;
   - que parte igual necesitaste revisar manualmente.

## Entregable

- `issue_borrador_sin_mcp.md`
- `issue_borrador_con_mcp.md`
- comparacion breve

# Clase 7: Customizacion avanzada de Copilot

## Objetivo

Usar custom instructions, prompt files, skills y MCP para convertir interacciones sueltas con Copilot en flujos mas consistentes, repetibles y mejor conectados con contexto y herramientas reales.

## Cambio de nivel respecto de las clases anteriores

- Clase 5:
  - integrar APIs y modelos en flujos concretos.
- Clase 6:
  - operar mejor esos flujos con Copilot CLI.
- Clase 7:
  - formalizar conocimiento reutilizable para que Copilot trabaje con menos ambiguedad y mayor consistencia.

## Resultados de aprendizaje

- Diferenciar prompt aislado, instruction, skill, MCP y agente.
- Identificar cuando conviene encapsular contexto e instrucciones reutilizables.
- Reconocer que aporta MCP y cuando no vale la pena usarlo.
- Usar skills desde agente y desde CLI en tareas acotadas.
- Reconocer beneficios y limites reales de la customizacion.

## Contenidos

- Rol de la customizacion en Copilot.
- Diferencia entre prompt, custom instructions, prompt files y skills.
- Beneficios y contras de usar skills.
- MCP:
  - que es;
  - que aporta;
  - que no aporta.
- Casos tecnicos donde la reutilizacion si aporta.
- Instalacion y prueba de skills y MCPs seleccionados.
- Comparacion entre prompt ad hoc, skill y tooling externo.

## Actividad practica

Instalar y probar algunos skills y MCPs, comparando su uso contra prompts ad hoc o tareas sin tooling externo.

## Entregables

- Comparacion breve entre prompt libre, skill o MCP segun el ejercicio.
- Observacion sobre donde la customizacion o el tooling aportaron y donde no.
- Lista corta de tareas del curso que podrian beneficiarse de customizacion reusable o contexto externo.

## Preparacion

- Tener acceso operativo a Copilot en el entorno usado durante la clase.
- Reutilizar casos o tareas tecnicas vistas en clases anteriores.
- Tener listos los MCPs seleccionados para demos o ejercicios.
